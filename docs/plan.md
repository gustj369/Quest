## 도메인 결정 메모

상세 정책은 [Product Decisions](./product-decisions.md)에 정리합니다.

### 미분류 퀘스트

현재 정책은 **새 퀘스트는 반드시 카테고리에 연결한다**입니다.

- 카테고리가 0개이면 퀘스트 추가 시 카테고리 생성 안내를 먼저 보여줍니다.
- 카테고리를 삭제하면 기존 퀘스트는 보존하고 `categoryId: null` 상태가 될 수 있습니다.
- `categoryId: null`은 데이터 보존을 위한 임시 상태로 보고, 새 퀘스트 생성 옵션으로는 노출하지 않습니다.

미분류를 정식 기능으로 열려면 아래 범위를 함께 설계합니다.

- 홈 화면에 `미분류` 탭 또는 필터 추가
- 카테고리 통계와 전체 카운트에서 미분류 표시 방식 결정
- 퀘스트 추가/수정 화면에서 미분류 선택 허용 여부 결정
- 추천 템플릿이 미분류 상태에서 어떻게 동작할지 결정
- 카테고리 삭제 시 기존 퀘스트를 미분류로 이동한다는 문구 보강

정식 미분류 탭은 사용자 흐름을 더 본 뒤 별도 작업으로 결정합니다.

### 푸시 알림 발송

현재 앱은 알림 권한, ON/OFF, 알림 시간을 저장합니다. 실제 발송은 아직 연결하지 않습니다.

정식 구현 전에 아래 정책을 결정합니다.

- 로컬 브라우저 알림만 지원할지, 서버 푸시까지 지원할지
- 앱이 닫혀 있을 때도 울려야 하는지
- Supabase 연동 전에도 사용자별 알림 구독을 저장할지
- iOS PWA와 Android Chrome의 지원 차이를 어떻게 안내할지

MVP에서는 “시간 설정 저장 + 권한 상태 안내”까지만 완료된 범위로 봅니다.
실제 발송 구현은 Supabase 또는 별도 서버 푸시 정책을 정한 뒤 진행합니다.

### 퀘스트 카드 밀도

긴 퀘스트 제목은 현재 한 줄 말줄임으로 처리합니다. 오른쪽에는 몬스터, 수정, 삭제 버튼을 하나의 고정 액션 영역으로 묶어 둡니다.

사용 중 긴 제목이 많아져 정보가 답답해지면 아래 순서로 조정합니다.

- 제목을 2줄까지 허용할지 검토
- 카테고리/난이도/XP 메타 정보를 더 작은 칩 구조로 정리
- 오른쪽 액션 영역 폭을 유지한 채 카드 높이를 조금 늘릴지 결정
