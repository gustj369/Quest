# Manual QA Checklist

Quest를 배포하거나 모바일 화면을 점검하기 전에 실제 기기에서 확인한 결과를 기록하는 문서입니다.
체크박스는 확인한 날짜와 간단한 메모를 함께 남깁니다.

## 2. 기본 화면 흐름

- [ ] 홈 화면이 모바일 폭에서 좌우 잘림 없이 보인다.
- [ ] 카테고리 화면으로 이동할 수 있다.
- [ ] 캐릭터 화면으로 이동할 수 있다.
- [ ] 하단 탭바가 화면 하단에 자연스럽게 고정된다.
- [ ] 스크롤 끝에서 주요 카드가 탭바에 가려지지 않는다.

메모:

## 3. 긴 제목 카드

아래 테스트 제목 5개를 직접 퀘스트로 추가해 확인합니다.

- 아침 출근 전에 물 2리터 마시고 텀블러 다시 채워두기
- 오늘 배운 영어 표현 10개를 예문까지 같이 복습하기
- 자기 전 10분 명상하고 오늘 감정 한 줄 기록하기
- 주간 가계부 정리하고 불필요한 지출 1개 줄이기
- 읽던 책 20페이지 읽고 핵심 문장 하나 저장하기

확인 항목:

- [ ] 제목 의미를 알아볼 수 있다.
- [ ] 카테고리, 난이도, XP 정보가 서로 겹치지 않는다.
- [ ] 몬스터, 수정, 삭제 버튼 영역이 너무 좁지 않다.
- [ ] 같은 유형의 긴 제목이 많아져도 카드 목록을 읽을 수 있다.

메모:

## 4. 미분류 퀘스트 흐름

카테고리 삭제 후 남은 퀘스트가 데이터 보존 정책대로 동작하는지 확인합니다.

1. 테스트 카테고리를 하나 만든다.
2. 해당 카테고리에 퀘스트를 하나 추가한다.
3. 카테고리를 삭제한다.
4. 홈 전체 탭에서 퀘스트가 보존되는지 확인한다.
5. 해당 퀘스트 수정 화면을 연다.
6. 카테고리 선택 안내가 보이는지 확인한다.
7. 새 카테고리를 선택해야 저장되는지 확인한다.

확인 항목:

- [ ] 카테고리를 삭제해도 연결된 퀘스트가 사라지지 않는다.
- [ ] 전체 탭에서 남은 퀘스트를 찾을 수 있다.
- [ ] 수정 화면에서 삭제된 카테고리 안내가 보인다.
- [ ] 새 카테고리를 선택하기 전에는 저장되지 않는다.
- [ ] 새 카테고리를 선택하면 정상 저장된다.

메모:

## 5. 바텀시트

새 퀘스트, 퀘스트 수정, 카테고리 추가, 설정 시트를 확인합니다.

- [ ] 열림 애니메이션이 끊기지 않는다.
- [ ] X 버튼으로 닫을 수 있다.
- [ ] 배경 영역 터치로 닫을 수 있다.
- [ ] 닫힘 애니메이션 중 배경과 시트가 어색하게 따로 움직이지 않는다.
- [ ] 닫힌 뒤 스크롤, 탭 전환, 버튼 터치가 정상 동작한다.

메모:

## 6. 퀘스트 완료와 캐릭터

- [ ] 퀘스트 완료 시 카드 애니메이션 후 완료 상태가 반영된다.
- [ ] XP가 증가한다.
- [ ] 완료 수가 증가한다.
- [ ] 레벨업 조건에서 레벨업 모달이 표시된다.
- [ ] 캐릭터 화면의 활동 기록과 뱃지가 크게 어긋나지 않는다.

메모:

## 7. PWA 설치

- [ ] manifest 아이콘이 DevTools Application 탭에서 보인다.
- [ ] 설치 배너 또는 브라우저 설치 메뉴가 표시된다.
- [ ] 홈 화면에 추가 후 앱처럼 실행된다.
- [ ] 설치 후 기본 화면이 정상 표시된다.
