# 작업 규칙 — Claude와 일하는 방식

## 소통 규칙
1. **한국어**로 설명하고 대화한다
2. 코드 변경 전 "무엇을 왜 바꾸는지" 한 줄 요약 후 진행
3. 작업이 끝나면 **스크린샷 또는 빌드 결과**로 확인
4. 오류 발생 시 원인을 먼저 분석하고, 수정 후 재확인

## 코드 작업 규칙
1. **기존 파일 수정**은 Edit 도구 사용, 전체 교체는 Write 도구
2. 새 컴포넌트 추가 시 `TEMPLATES/new-feature-template.md` 참고
3. 컬러·폰트는 `CLAUDE.md`의 디자인 시스템 기준 준수
4. `node_modules/`, `dist/`는 절대 건드리지 않음

## 디자인 규칙
1. Press Start 2P → 제목·레벨·수치에만 (짧은 텍스트)
2. Noto Sans KR → 본문·퀘스트명·설명 (가독성 우선)
3. CSS 변수(`--accent-primary` 등)를 하드코딩 색상보다 우선
4. 터치 타겟 최소 44px 이상 유지

## 브랜치/커밋 규칙
1. 기능 단위로 커밋 (작업 완료 후 요청 시)
2. 커밋 메시지 한국어 + 영어 병행 가능
3. 배포는 Vercel (요청 시 진행)

## 금지 사항
- 보라색 그라디언트, Inter 폰트 등 제네릭 AI 디자인
- `console.log` 남기기 (디버그 후 반드시 제거)
- 한 번에 너무 많은 파일 동시 수정 (추적 불가)
- TypeScript 강제 도입 (사용자 요청 전까지 JS 유지)
